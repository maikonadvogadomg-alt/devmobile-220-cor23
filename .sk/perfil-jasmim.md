# Perfil Jasmim — Notas de Trabalho

## Sobre Saulo
- Saulo é advogado.
- Tem limitação nos membros superiores.
- Usa muito comando de voz.
- Prefere respostas claras, diretas e em português simples.
- É importante evitar jargão técnico sem explicar.

## Sobre o projeto
- O objetivo não é só “ter um app bonito”.
- O foco real é transformar o DevMobile em uma IDE mobile local-first.
- O celular deve ser o centro da operação.
- O app deve funcionar localmente mesmo sem Replit.
- Termux pode ser uma melhoria, mas não pode ser obrigatório.
- VS Code-like, Monaco e outros modos devem ser fallback, não dependência de servidor Replit.

## Preferências de execução
- Quando o terminal falha, é melhor documentar primeiro.
- Mudanças grandes devem ser feitas em blocos seguros.
- O diário do projeto deve sempre refletir a situação real, sem fingir que algo já foi corrigido no código quando ainda não foi.

## Direção técnica consolidada
- Remover acoplamento ao Replit.
- Centralizar base URL e runtime.
- Separar UI de execução.
- Priorizar persistência local.
- Tornar IA, GitHub, banco remoto e terminal avançado recursos opcionais.
